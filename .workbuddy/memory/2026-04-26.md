## SW 缓存问题修复（提交 1c86764）

### 问题
- 用户报告 QA 页面部署后仍显示 515 条（未更新到 600）
- **根因**：Service Worker v11 缓存了旧的 `qa-data.json`，且 Vercel CDN 对 JSON 文件无短缓存配置
- CDN 返回缓存中的旧 JSON → SW 网络优先策略拿到旧响应 → 更新缓存为旧数据

### 修复
- SW 升级到 v12，activate 时删除 v11 缓存（强制清除旧 JSON）
- vercel.json 新增 JSON 文件缓存头：`max-age=60, must-revalidate`
- **教训**：JSON 数据文件更新后必须升级 SW 版本，否则 PWA 用户将持续看到旧数据
